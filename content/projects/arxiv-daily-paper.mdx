## 痛点

读博每天刷论文，ArXiv 上新几百篇，挨个翻标题太费精力。

市面工具要么不聚焦我的方向，要么只给英文邮件。我想要的很简单：

> **中文摘要 + 微信推送 + 关键词可定制。** 那就自己写一个。

---

## 做了什么

全自动论文监控速递系统，每天从三个来源抓取最新论文：

**数据源**
- **ArXiv** — cs.CV、cs.AI、cs.LG 等类别当日更新
- **Crossref** — 城市遥感、GIS 相关期刊最新文章
- **Semantic Scholar** — 补充高影响力论文，补全缺失摘要

**筛选流水线（四级）**

```
① 关键词打分 ── 50+ 自定义关键词（U-Net、Transformer、时空融合…）
② DeepSeek 精筛 ── 大模型判断与"城市遥感 + 时空智能"的真实相关度
③ 摘要补充 ── 从多源拉取补全每篇论文摘要
④ LLM 摘要 ── 每篇生成一句白话中文总结
```

**推送渠道**

```
📱 微信公众号图文 ── 起床第一眼看到
📧 QQ 邮箱 ── 备选存档
🤖 飞书机器人 ── 推送到项目协作群
```

## 技术栈

| 层级 | 技术 |
|---|---|
| **语言** | Python 3.13 |
| **数据源** | ArXiv API / Crossref API / Semantic Scholar API |
| **大模型** | DeepSeek（相关度判断 + 中文摘要生成） |
| **存储** | SQLite（推送历史去重、论文状态管理） |
| **推送** | 微信公众号图文素材 / QQ 邮箱 SMTP / 飞书 Webhook |
| **定时** | Windows 任务计划程序 |

---

## 日常使用

代码跑在本地 Windows 上，每天定时触发，零维护：

```bash
# 每天早上 8:00 自动执行，不用管
python main.py

# 预览当天结果但不推送
python main.py --dry-run

# 跳过 DeepSeek 快速预览
python main.py --skip-llm --dry-run
```
