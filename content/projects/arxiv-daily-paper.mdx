## 为什么要做这个

读博期间每天要刷大量论文，但 ArXiv 每天上新几百篇，挨个看标题摘要太耗精力。我需要一个**自动筛选器**——把和我研究方向相关的论文挑出来，每篇配一句人话总结，每天早上推送到手机上。起床看一眼微信，就知道今天该读哪篇。

市面上有 Paper Digest、Connected Papers 之类的工具，但它们要么不聚焦我的方向，要么只给英文邮件。我想要的很简单：**中文摘要 + 微信推送 + 关键词可定制**。那就自己写一个。

## 做了什么

一个全自动的论文监控与速递系统，每天从三个来源抓取最新论文：

- **ArXiv** — 按 cs.CV、cs.AI、cs.LG、stat.ML 等类别拉取当日更新
- **Crossref** — 搜索城市遥感、GIS 相关期刊的最新文章
- **Semantic Scholar** — 补充高影响力论文，补全跨库论文的缺失摘要

然后经过**四级筛选流水线**：

1. **关键词打分** — 用 50+ 个自定义关键词（U-Net、Transformer、遥感、时空、地理空间…）给每篇论文打分
2. **DeepSeek 精筛** — 候选论文过一遍大模型，判断和"城市遥感 + 时空智能"方向的真实相关度，再排序
3. **摘要补充** — 缺摘要的论文从 Semantic Scholar/Crossref 拉取，确保每篇都有摘要可用
4. **LLM 摘要** — 给最终选中的论文生成一句话白话中文总结

最后渲染成三种格式推出去：
- 📱 **微信公众号图文** — 起床第一眼看到
- 📧 **QQ 邮箱** — 备选渠道，可以保留存档
- 🤖 **飞书机器人** — 推送到项目协作群

## 我每天怎么用

不需要打开任何网页。每天醒来，微信订阅号里就有今天的日报：

> **「城市遥感与 GIS 论文速递 | 05-10」**
>
> 本期精选 3 篇论文，涵盖方向：城市遥感、时空智能、多源数据融合
>
> 1️⃣ From Drops to Grid — 用神经过程融合稀疏站点与雷达数据做高分辨率降雨估计
> 2️⃣ Exponential distance decay — 美国 20 大都市区的公园访问距离衰减规律
> 3️⃣ ReTrEAT cities — 海平面上升背景下沿海社区韧性框架

点击文章链接直达原文。看到感兴趣的，白天在电脑上细读。

代码跑在本地 Windows 上，每天定时任务触发：

```bash
# 每天早上 8:00 自动执行，啥也不用管
python main.py
```

如果需要预览当天结果但不推送：

```bash
python main.py --dry-run
```

## 技术栈

| 层级 | 技术 |
|---|---|
| **语言** | Python 3.13 |
| **数据源** | ArXiv API / Crossref API / Semantic Scholar API |
| **LLM** | DeepSeek（相关度判断 + 中文摘要生成） |
| **数据存储** | SQLite（推送历史去重、论文状态管理） |
| **推送渠道** | 微信公众号（图文素材接口）/ QQ 邮箱（SMTP）/ 飞书机器人（Webhook） |
| **定时触发** | Windows 任务计划程序 |
